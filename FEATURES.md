## 1. 文件格式支持

| 功能 | 状态 | 说明 |
|------|------|------|
| OLE2 / CFB 复合文档解析 | ✅ 已实现 | 完整的 FAT/DIFAT/目录表解析 |
| Word 97-2003 二进制格式 | ✅ 已实现 | 支持主流 .doc 文件 |
| Word 6.0 / 95 格式 | ⚠️ 部分支持 | 可提取文本，格式支持有限 |
| UTF-16LE（Unicode）编码 | ✅ 已实现 | 完整支持 |
| 8-bit 压缩编码 | ✅ 已实现 | 支持 fComplex 标志位检测 |
| 编码自动检测 | ✅ 已实现 | 基于二进制模式 + 评分机制 |
| macOS textutil 生成的 .doc | ⚠️ 部分支持 | fComplex 标志不可靠，使用自动检测兜底 |
| .dot 模板文件 | ✅ 已实现 | 扩展名白名单支持 |
| DOCX (Open XML) | ❌ 不支持 | 建议使用专门的 DOCX 解析器 |

## 2. 文本内容

| 功能 | 状态 | 说明 |
|------|------|------|
| 纯文本提取 | ✅ 已实现 | 支持 `parse()` 方法获取纯文本 |
| 多字节字符（中文/日文/韩文） | ✅ 已实现 | 完整支持 CJK 字符 |
| 特殊符号 | ⚠️ 部分支持 | 支持常见标点、引号等，部分符号可能遗漏 |
| 换行符（段落标记） | ✅ 已实现 | 正确识别 0x0D 段落标记 |
| 分页符 | ❌ 待实现 | 当前合并为普通段落 |
| 分节符 | ❌ 待实现 | 当前合并为普通段落 |
| 制表符（Tab） | ⚠️ 部分支持 | 渲染时可能未正确对齐 |
| 软换行（Shift+Enter） | ❌ 待实现 | 手动换行符可能未正确处理 |
| 不间断空格 | ❌ 待实现 | 可能被当作普通空格 |
| 可选连字符 | ❌ 待实现 | |
| 文本框内容 | ❌ 待实现 | 文本框中的文字当前可能提取不到 |

## 3. 字符格式

| 功能 | 状态 | 说明 |
|------|------|------|
| 字体名称 | ⚠️ 启发式检测 | 基于 CJK/ASCII 字符分布推断，不读取 CHP 表 |

| 斜体（Italic） | ❌ 待实现 | 未检测 |
| 下划线（Underline） | ⚠️ 启发式检测 | 部分场景可识别 |
| 删除线（Strikethrough） | ❌ 待实现 | |
| 文字颜色 | ❌ 待实现 | |
| 高亮（Highlight） | ❌ 待实现 | |
| 上标 / 下标 | ❌ 待实现 | |
| 字符间距 | ❌ 待实现 | |
| 字符边框 / 底纹 | ❌ 待实现 | |
| 隐藏文字 | ❌ 待实现 | 默认不提取隐藏文字 |

> **说明**：当前格式检测基于**启发式规则**（文本特征推断），未真正解析 Word 的 CHP（Character Properties）表。未来可扩展为真实格式解析。

## 6. 页眉页脚

| 功能 | 状态 | 说明 |
|------|------|------|
| 页眉（Header） | ❌ 待实现 | 当前只提取主文档内容，页眉被跳过 |
| 页脚（Footer） | ❌ 待实现 | 同上 |
| 首页不同 | ❌ 待实现 | |
| 奇偶页不同 | ❌ 待实现 | |
| 页码域（PAGE / NUMPAGES） | ❌ 待实现 | |
| 页眉页脚中的图片 | ❌ 待实现 | |

> **说明**：Word 文档包含多个 story（主文档、页眉、页脚、脚注、尾注等）。当前解析器只读取第一个 piece（主文档），其余 story 均被跳过。这是一个设计选择——避免页眉页脚内容混入正文。未来可作为可选功能提供。

## 🎯 优先级路线图（建议）

### 高优先级（用户体验提升明显）

1. **表格支持** - 表格是常见文档元素，当前平铺可读性差
2. **页眉页脚（可选展示）** - 解决带页眉页脚文档的兼容性问题
3. **超链接支持** - 提升交互体验
4. **图片提取与展示** - 图文混排文档的刚需

### 中优先级（格式还原度）

5. **真实 CHP/PAP 格式解析** - 替代启发式检测，提高格式准确率
6. **文档属性读取** - 显示标题/作者/创建时间等元信息
7. **目录生成** - 自动生成文档大纲导航

### 低优先级（锦上添花）

8. **脚注/尾注**
9. **批注显示**
10. **更多域支持**
